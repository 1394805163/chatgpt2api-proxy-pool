# v1.6 选择性升级实现计划

> **面向 AI 代理的工作者：** 使用 test-driven-development 逐项执行，先观察失败测试，再写最小实现。

**目标：** 在不覆盖本地增强的前提下，将官方 v1.6.0 中更优且缺失的能力适配到定制分支。

**架构：** 不合并官方提交历史，而是围绕现有服务边界移植行为。后端以幂等资源释放和批量接口为核心，前端新增的解析逻辑保持为可测试纯函数。

**技术栈：** Python 3.13、unittest、FastAPI、curl_cffi、Next.js 16、React 19、TypeScript、Bun、Docker。
