# v1.6 选择性升级设计

## 目标

以官方 `v1.6.0` 为升级来源，将本地尚未具备且确实更优的能力移植到定制分支，同时保留注册机、Sentinel SDK、并发生图、持久化、额度控制及账号池策略等本地增强。

## 版本边界

- 当前本地基线：`1.5.0 番外篇3.0`。
- 目标标识：`1.6.0 番外篇4.0`。
- 官方 `v1.7.0` 删除注册模块，不进入本次升级范围。
- 不直接 merge 或 rebase 官方分支，所有改动按行为逐项移植。

## 采用的官方能力

### Session 生命周期

为 `OpenAIBackendAPI` 增加幂等 `close()` 和上下文管理协议，并在账号刷新、文字流、搜索、模型列表、图片任务、可编辑文件任务等短生命周期调用点保证释放。流式响应必须先消费或关闭，再释放 Session。

### 账号 JSON 导入

账号导入页同时支持单账号对象、账号数组，以及 `accounts` 或 `items` 包装结构。保留本地现有的异步刷新进度、重复账号跳过和 CPA 兼容逻辑。

### 图片队列空结果恢复

当一个 turn 已无 loading 图片但状态仍是 queued/generating 时，根据实际图片结果收敛为 success/error，避免旧状态占据队列或页面持续显示排队。适配当前按 turn 并发的实现，不照搬官方按 conversation 串行队列。

### Sub2API 批量导入

列表展示不再依赖列表响应里存在 access_token；导入改用 `/api/v1/admin/accounts/data?ids=...` 一次导出所选账号，并保留后台任务状态及逐账号错误汇总。

### 邮箱验证码边界

邮箱创建时记录验证码接收起点，忽略更早的历史邮件。CloudMailGen 创建地址时调用 `addUser`，确保后端已登记收件地址。

### 上游对话清理

增加“生图完成后删除 ChatGPT 上游对话”开关，默认关闭。仅删除 ChatGPT 上游会话，不删除本地图片、系统日志、浏览器历史或数据库记录。

## 保留的本地实现

- 推理强度透传已覆盖 Chat Completions、Responses、缓存键和调试页。
- 注册邮箱代理与 CloudMailGen token 失效刷新已实现。
- Web、图片和缩略图的 HEAD 路由覆盖比官方更完整。
- Pro/ProLite 未知额度保持可用，不采用官方统一有限额度策略。
- Sentinel SDK、SO token、5000 ms observer、注册 authorize/continue 保持不变。
- 生图总时限、任务 ID、并发请求、图片持久化、日志持久化和用户额度保持不变。
